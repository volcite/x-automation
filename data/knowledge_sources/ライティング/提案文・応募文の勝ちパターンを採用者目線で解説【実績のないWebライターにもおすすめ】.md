## Summary
Webライターの案件獲得において、クライアントの採用判断の基準となる「提案文」の書き方、実績がない場合の突破戦略、そして継続案件につなげるためのクライアントワークのコツについて解説している。提案文では「自分視点」から「クライアントへの貢献視点」に変えることが重要であり、採用されやすい構成や実績の見せ方、初心者向けのサンプル記事作成方法、信頼を得るためのコミュニケーション術（即レスや進捗報告など）を具体的に紹介している。

## Key Learning Points
*   **提案文の本質と重要性**
    *   クライアントはスキルを直接確認できないため、提案文で「任せて大丈夫か」を判断する。
    *   主語を「自分が稼ぎたい」から「クライアントの課題解決・利益貢献」に変えることが必須。
    *   クライアントの真の目的（売上、採用など）を理解し、不安（納期、品質、コミュニケーション）を取り除く提案を行う。
*   **提案文の勝ちパターン（構成の型）**
    1.  **件名**: 隅付き括弧で強みを入れる（例：【SEO実績あり】〇〇案件への応募）。
    2.  **挨拶・自己紹介**: 案件に関連する経歴のみを端的に記載。
    3.  **案件内容の確認**: 募集要項を読んでいることを示すため、内容を復唱して承知した旨を書く。
    4.  **経歴・専門性**: 案件に直結する職歴や資格（証券会社勤務、FPなど）を記載。無関係なものは省く。
    5.  **実績・ポートフォリオ**: 数字（本数、PV数）で示し、記事URLを添付して証拠を見せる。
    6.  **具体的な提案**: 「記事が書ける」だけでなく、構成作成、入稿、画像選定、リライト提案などプラスアルファを提示する。
    7.  **稼働条件**: 具体的な数字（週20時間、24時間以内に返信など）で示し、安心感を与える。
    8.  **締めの挨拶**: シンプルに記載。
*   **実績不足の突破戦略**
    *   **ポジティブな言い換え**: 「初心者です」と言わず、「実務未経験だが基礎スキル（SEO/WordPress）は習得済み」などと伝える。
    *   **自主制作による可視化**: 応募ジャンルのキーワードで3000〜5000文字のサンプル記事を自作し（Googleドキュメントやnote）、URLを提示する。
    *   **戦略的な低単価**: 最初の5〜10件は評価を稼ぐために低単価（0.5円前後）で受け、実績を作ってから適正価格に戻す。ただし品質は落とさない。
*   **継続案件の獲得方法**
    *   期待値を少しだけ（101点）上回り続ける意識を持つ。
    *   **即レス**: 連絡には1時間以内、遅くとも当日中に返信する。
    *   **進捗報告**: クライアントに聞かれる前に、区切りの良いタイミングで自分から報告する（マネジメントコスト削減）。
    *   **納期前の納品**: 納期を守るのは当然。余裕を持って早めに納品することで評価が上がる。

## Key Message
Webライターとして案件を獲得し、継続して依頼されるためには、クライアントの不安を取り除き、利益に貢献できるビジネスパートナーであることを、提案文や日々の迅速なコミュニケーションを通じて証明し続けることが重要である。
