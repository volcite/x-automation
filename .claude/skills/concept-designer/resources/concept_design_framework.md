# コンセプト設計フレームワーク

## 1. 3C分析

### Customer（顧客）
- **誰が買うのか？** 年齢、性別、職業、年収、家族構成
- **何に悩んでいるのか？** 表面的な悩みと根本的な悩み
- **今どんな解決策を使っているか？** 現状の代替手段
- **理想の未来は？** 手に入れたい結果・感情

### Competitor（競合）
- **直接競合**: 同じ商品・サービスを提供している相手
- **間接競合**: 同じ悩みを別の方法で解決している相手
- **競合の強み**: 価格、機能、ブランド、実績
- **競合の弱み**: サポート不足、高額、難しい等

### Company（自社）
- **自分の強み**: 実績、経験、独自のノウハウ
- **自分の弱み**: 知名度、リソース、価格
- **なぜ自分がやるのか**: 原体験、情熱、使命感

---

## 2. USP（独自の売り）抽出法

USPは以下の3つの交点にある：

```
[顧客が求めていること]
       ∩
[自分にしかできないこと]
       ∩
[競合がやっていないこと]
```

### USP発見のための質問
1. お客様が「あなたから買う理由」は何ですか？
2. 競合と比べて「これだけは負けない」ことは何ですか？
3. お客様が他では得られない体験・結果は何ですか？

---

## 3. ポジショニングマップ

以下の軸で自社と競合を配置する：

### 一般的な軸の候補
- 価格（安い ←→ 高い）
- 手軽さ（簡単 ←→ 本格的）
- カスタマイズ性（汎用 ←→ 個別対応）
- サポート（セルフサービス ←→ フルサポート）
- 対象レベル（初心者向け ←→ 上級者向け）

### 選び方
- 自分の強みが活きる軸を選ぶ
- 競合がいない「空白地帯」を狙う
- ターゲットが重視する軸を選ぶ

---

## 4. コンセプト言語化テンプレート

### 一文コンセプト
「[ターゲット]が、[手段]で、[期間]で[結果]を手に入れる」

### エレベーターピッチ
「[ターゲット]は[問題]で困っている。[商品名]は[USP]によって[結果]を実現する。[競合]と違い、[差別化ポイント]が特徴」

---

## 5. コンセプト検証チェックリスト

- [ ] ターゲットが聞いて「私のことだ」と思うか？
- [ ] 一言で何の商品か伝わるか？
- [ ] 競合と明確に違いがあるか？
- [ ] 価格に見合う価値を感じるか？
- [ ] お客様の言葉で語れているか？（専門用語を使っていないか）
- [ ] 「だから何？」と言われない具体性があるか？
