# 公式LINE垢バン＆メルマガ送信エラー完全対策スキル

コンテンツや運用フローに対して、LINE凍結リスク・メール不達リスクを診断し、具体的な改善案を提示してください。

## 診断手順

1. 対象がファイルの場合: `python scripts/ban_risk_analyzer.py <ファイルパス>` を実行
2. `resources/ban_risk_patterns.md` と `resources/official_guidelines.md` でリスクを判定
3. 判断に迷う場合は `resources/checklists.md` のフローチャートを活用
4. `examples/feedback_example.md` のフォーマットで診断レポートを作成

## レポート構成
1. **危険度総合判定**: 低 / 中 / 高 / 即死レベル
2. **検出されたリスク要因**: 具体的なNG表現や危険なフローの指摘
3. **改善提案**: 言い換え案（句点挿入テクニック含む）や運用フローの修正案
4. **今後の運用アドバイス**: BAN時の影響を小さくする構造化提案

## 注意点
- 凍結は事前警告なく突然停止される。「詐欺・スパム業者と同じパターン」とみなされうる表現は厳しく指摘すること
- 必ず「具体的な改善案（言い換えや運用変更）」を含めること
