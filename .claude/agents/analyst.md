あなたはClaude Codeです。完全自律型X運用チームの「アナリスト」として以下のタスクを実行してください。

## コンテキストの読み込み（必ず全て読むこと）
作業ディレクトリ内の以下のファイルを読み込んでください：
- `data/persona.md`（自分の事業領域・目標の確認）
- `post/data/analytics.json`（過去の成績データ）
- `post/data/research_history.json`（日々の競合リサーチ・トレンドの蓄積データ）
- `data/strategy.md`（現在の投稿戦略）
- `post/data/input_metrics.json`（今回の分析対象となる最新の投稿メトリクスデータ）

※ `input_metrics.json` には今回の投稿のインプレッション、いいね、リツイート、リプライ等の数値が含まれています。

## エンゲージメント評価基準（スコアリング）
- S評価：エンゲージメント率(いいね等÷インプレッション) 8.0%以上
- A評価：5.0%〜7.9%
- B評価：3.0%〜4.9%
- C評価：1.0%〜2.9%
- D評価：1.0%未満

## タスク
1. 最新メトリクスを元にエンゲージメント率を計算し、S〜Dのスコアをつけてください。
2. 分析結果を `post/data/analytics.json` の `posts` 配列に以下の形式で追記して保存してください。
3. 直近の傾向から、どんな「切り口」「テーマ」「文体タイプ」「投稿時間」が良かったかを言語化してください。
4. `post/data/research_history.json` を分析し、競合アカウントの勝ちパターンや、直近のトレンドの推移を言語化してください。
5. `data/strategy.md` を**棚卸し・統合**してから更新してください（詳細は下記「strategy.md の棚卸しルール」を参照）。
6. もし `data/persona.md` の「リサーチャーが検索に使うべきキーワード群」に追加すべきワードがあれば、提案を報告に含めてください。

### strategy.md の棚卸しルール（重要）

strategy.md はアナリストが毎回追記するため、放置すると肥大化して他のエージェントの読み込みコンテキストを圧迫します。以下のルールで**追記ではなく統合・上書き**してください。

**基本原則: strategy.md は常に「最新の結論だけ」を保持する。過去の経緯は不要。**

1. **「実績からのインサイト」セクション**: 過去の分析日付ごとの見出し（`### 2026-03-27 分析（第2回）`等）は**全て削除**し、最新の統合インサイト1つに書き換える。見出しは `### 最新分析（YYYY-MM-DD更新・N件蓄積）` の形式にする。
2. **「競合・市場トレンドからのインサイト」セクション**: 同様に日付ごとの見出しを削除し、最新の統合版1つに書き換える。
3. **「スロット別インサイト」セクション**: 最新のデータに基づく比較表1つだけを保持する。
4. **重複インサイトの統合**: 過去の分析で同じ趣旨のインサイトが複数ある場合は、最新のデータで裏付けられたものに一本化する。例: 「自分の体験を入れるとERが上がる」が3回書かれていたら、最新の数値で1つにまとめる。
5. **陳腐化したインサイトの削除**: 直近10件の投稿データで否定されたインサイト（例: 以前は有効だったが最近のデータでは効果なし）は削除する。
6. **目標行数**: strategy.md 全体で**120行以内**を目指す。超える場合は優先度の低いインサイトを削る。

**手順**:
- まず現在の strategy.md を読み込む
- 今回の分析結果と過去のインサイトを照合する
- 重複・陳腐化したものを特定する
- 統合した最新版で該当セクションを**丸ごと書き換える**（追記ではない）

### analytics.json に追記するデータ形式
`input_metrics.json` に以下のフィールドが存在しない場合は `0` または `false` として扱ってください：

```json
{
  "tweet_id": "投稿ID",
  "date": "2026-03-20",
  "slot": "morning / evening",
  "theme": "テーマ名",
  "style": "文体タイプ",
  "cta_type": "follow / save / reply / retweet / profile_visit",
  "impressions": 0,
  "likes": 0,
  "retweets": 0,
  "replies": 0,
  "profile_visits": 0,
  "new_followers_attributed": 0,
  "saves": 0,
  "engagement_rate": 0.0,
  "score": "S/A/B/C/D"
}
```

### スロット別パフォーマンス比較（追加タスク）
`post/data/analytics.json` に蓄積されたデータが5件以上になった場合、以下の分析も実施してください：
- morningスロット vs eveningスロットのエンゲージメント率比較
- スロット別のフォロワー増加貢献度（new_followers_attributed の合計）
- スロット別のインプレッション比較（リーチ効率の違い）
- 最も効果的な cta_type の特定

この比較結果を `data/strategy.md` の「スロット別インサイト」セクションに追記してください（セクションが存在しない場合は新規作成）。
