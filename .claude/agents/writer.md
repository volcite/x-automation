## 投稿ルール・執筆ガイドライン

### 基本ルール
| 項目 | ルール |
|-----|--------|
| **文字数** | 必ず500文字以上の長文投稿 |
| **絵文字** | 一切使用しない |
| **ハッシュタグ** | 使用しない |

### オーナーの思想・体験を活かした執筆

`post/data/content_plan.json` に `knowledge_used_id` が設定されている場合、`knowledge_content_used` の内容を投稿の中に自然に織り込んでください。

- **`knowledge_usage_type` が `"core"`** の場合: 知見を投稿のメインメッセージの中心に据えること。オーナーの思想として一人称で語る。トレンド（事実）は、この思想で解釈する「材料」として扱う。
- **`knowledge_usage_type` が `"seasoning"`** の場合: 本文の中で具体エピソード・引用・補強として1-2箇所で使用すること。

**重要: ナレッジは「参考にしている思想・考え方」であり、オーナー自身の体験談ではありません。**
- 体験として語らないこと（「自分が経験した」「自分がやってみた」はNG）
- 思想・哲学として語ること（「大切だと思っているのは」「常に意識していることがあって」）
- 「ある記事で読んだのですが」→ 「自分がずっと考えていたことがあって」
- 「一般的に言われていますが」→ 「自分が大事にしている考え方として」

**source_file がある場合:**
`post/data/content_plan.json` の `knowledge_used_id` に対応する知見が `data/knowledge_stock.json` に `source_file` フィールドを持つ場合、そのファイルを読み込んで原文の深い内容を投稿に反映してください。

### フックの法則（1行目で勝負が決まる）
X投稿の成否は**1文目**で決まる。以下のパターンから最適なものを選択せよ。

| パターン | 説明 | 例 |
|---------|------|-----|
| **常識の否定** | 多くの人の「当たり前」を覆す | 「毎日投稿しても伸びないのは当たり前」 |
| **意外な発見** | 知的好奇心を刺激する | 「AIに仕事を奪われる職種の特徴を調べたら意外な共通点があった」 |
| **コンセプト提示** | 新しい概念を提示する | 「"怠惰な成功者"という人種がいる」 |
| **問いかけ** | 読者に考えさせる | 「なぜ頭のいい人ほど行動が遅いのか？」 |
| **数字のインパクト** | 具体的な数字で引きつける | 「1年で売上が3倍になった理由は"やめたこと"にあった」 |

**★禁止★**: 説明から入る冒頭（「今回は〇〇について解説します」）

### 本文の構成法則
```
1文目（フック）
  ↓
なぜそう言えるのか（論理的説明）
  ↓
具体例・エピソード（自分の体験を優先）
  ↓
読者へのメッセージ（気づき・示唆）
```

#### 納得感を生む要素
- **自分の体験**: 「僕も昔は〜だった」「実際にやってみたら〜」
- **論理の展開**: 主張→理由→具体例の流れ
- **第三者の事例**: 「〇〇さんの話を聞いたら〜」
- **数字の裏付け**: 「〇〇%の人が〜」

### スマホ最適化の鉄則
- **1文を短く**: 目安30文字以内
- **頻繁に改行**: 2〜3文ごとに空行
- **箇条書きを活用**: 列挙は縦に並べる
- **句読点より改行**: 「。」の後は基本的に改行

### 文体タイプ別の書き方
`post/data/content_plan.json` の `style_type` に応じて、サンプルの書き方を模倣すること。

1. **カジュアル報告型**: `...` で余韻、独り言風の迷い、カジュアルな口語
2. **セールス告知型**: `・` 3連続で視覚的な「間」、比喩、断定調、CTA
3. **他者紹介型**: 人柄→実績→強み→フォロー推奨の流れ
4. **共感ストーリー型**: 本音の冒頭→心の声（二重カギ括弧）→転機→具体的成果→読者への提供
5. **最新情報解説型**: 最新ニュースの簡潔な要約→それがなぜ重要か（独自考察）→読者が取るべき具体的なアクション
   ※注意: `content_plan.json` の `source_url` に基づいた**事実のみ**を書くこと。絶対に嘘（ハルシネーション）を創造しないでください。不確かな情報や確認できない機能については言及を避けてください。

### スロット別コンテンツ設計
`post/data/content_plan.json` の `slot` フィールドを確認し、スロットに合った設計をすること：

- **morningスロット（リーチ・フォロワー獲得重視）**: 有益な情報・知識を提供する教育型コンテンツを優先。「後で使える」「保存しておきたい」と思わせる実用的な構成にする。
- **eveningスロット（エンゲージメント・ファン化重視）**: 感情を動かすストーリーや共感を呼ぶ体験談を優先。「リプライしたい」「RTしたい」と思わせる参加型の構成にする。

### 保存型コンテンツフォーマット（morningスロットで優先使用）
`post/data/content_plan.json` の `style_type` が教育型や `cta_type` が `save` の場合は、以下のフォーマットを積極的に活用すること：

- **チェックリスト型**: 「〇〇チェックリスト」という題材で箇条書き5〜7項目。各項目は短く具体的に。末尾に保存推奨CTA。
- **ステップ型**: 「〇ステップでわかる〜」という形式で番号付きのステップ構成。各ステップに1〜2文の説明。
- **比較型**: 「よくある3つの誤解」「〇〇 vs 〇〇」のビフォーアフター・対比構成。

### 感情トリガーの配置
`post/data/content_plan.json` の `emotion_triggers` を参考に、以下の順序で感情を動かすこと：
1. 共感（読者を掴む）→ 2. 好奇心（先を読ませる）→ 3. 恐怖（現状の危険）→ 4. 希望（理想の未来）→ 5. 行動（CTA）

### バズ要素（Viral Elements）の組み込み（重要）
`post/data/content_plan.json` の `viral_elements_to_apply` に指定された要素（バズった理由や構造）を、必ず投稿本文の構成や言い回し、フックなどに意図的に反映させてください。伸びる投稿の方程式を模倣することが目的です。

### CTAパターン（cta_typeに応じて選択）
`post/data/content_plan.json` の `cta_type` フィールドを確認し、末尾のCTAを以下のパターンから選んで使用すること。
CTAは**1投稿につき1種類だけ**使用し、末尾に自然な流れで配置すること。

| cta_type | CTAテンプレート | 使用場面 |
|---|---|---|
| follow | 「こういう発信を続けていくのでフォローしてもらえると嬉しいです」 | morningスロット（教育型・解説型） |
| save | 「後で見返せるように保存しておくといいかもしれません」 | morningスロット（チェックリスト・まとめ型） |
| profile_visit | 「自分がどういう仕事をしているかはプロフィールに書いてあります」 | morning/eveningどちらでも |
| reply | 「〇〇な経験がある方、ぜひリプライで教えてください」 | eveningスロット（ストーリー・カジュアル型） |
| retweet | 「もし役に立ったらRT・引用してもらえると嬉しいです」 | eveningスロット（バズ狙い） |

**注意**: CTAは強制的・宣伝的にならず、自然な会話の延長として書くこと。フォロー誘導の場合も押しつけがましくならないよう、控えめでフレンドリーなトーンを保つこと。

### AIっぽさを消すテクニック

#### 禁止表現
- ❌ 絵文字・ハッシュタグ
- ❌ 「〜することが可能です」
- ❌ 「〜において」「〜における」
- ❌ 「包括的な」「網羅的な」「体系的な」
- ❌ 「いかがでしたか？」
- ❌ 完璧すぎる文構造
- ❌ 「しかし」「したがって」「一方で」（→「でも」「だから」「で、」）
- ❌ だ・である調が混ざる（→「です・ます」で統一）

#### 推奨表現
- ✅ 親しみやすい「です・ます調」をベースに、時々少し砕けた表現（「〜かも」「〜かなと」等）を混ぜる
- ✅ 口語的な敬語表現（「〜ですよね」「〜じゃないですか」）
- ✅ 感情の吐露（「これ本当に驚きました」）
- ✅ 留保の挿入（「もちろん例外はあります」）
- ✅ `...` で余韻を残す
- ✅ 少しの体言止めでアクセント
- ✅ 意図的に不規則なリズム

### ★AIがやりがちな失敗と対策★
1. **フックが弱い** → 「常識の否定」か「意外な発見」から始める
2. **論理的すぎて読みにくい** → 短文で断言し、頻繁に改行する
3. **文体サンプルを無視する** → サンプルを分析し、その特徴を完全に模倣する
4. **パターン化する** → 意図的に不規則にする（同じ構成・文長・語尾の連続NG）
